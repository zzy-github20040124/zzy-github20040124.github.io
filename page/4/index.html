hexo的困难

​ 第一次搭建博客真的遇见了好多困难，一次次的报错真的叫人崩溃，还好现在成熟的网络想找到答案也不是一件困难事。

1.fatal port 4000 has been used.这个代表你用的域名你现在还不能用。

可以等到一个可以用的时机或者换一个域名(使用命令hexo server -p 加上你想更改的端口)

2.can not read a block mapping.（这是一个很小的错误但是我们往往会忘记）

在配置文件里面，都是按照属性名+英文冒号+英文空格+属性值，这个形式进行属性设置的，所以千万不要忘记空格不然报错真的很心累。

（我确实也没想到hexo能使用HTML中的语法markdown，倒是很好用）

3.困阻我最久的当属hexo的部署问题在下属文件夹中要改的东西不多，但众说纷纭各有各的说法我挨个摸索确实耗费了大量精力。如果有部署方面的问题可以来问我vx：zzy18771857592，当然也欢迎各位加我共同成长。

关于自学web的一些笔记

自己在学HTML中的一点笔记和心得。

1.图片导入— 找到图片保持存在同一个文件夹，可以直接img scr=‘’图片名称‘’。如果在子文件夹返回上一级img scr=‘../xxx’再输入图片所在文件夹。（用软件img scr=’‘可选路径’‘）

2.网站的跳转 — <a href=’’想要跳转的网站’’>放入信息（可以是文字可以是图片）title=’鼠标悬停上去后的提示信息’ target=’规定在何处打开文档‘</a>

3.<from method=’get或者post’ action=’向何处发送表单数据’>

​ <input type=’定义输入框类型‘>是输入框</input>

</from>

4.在外部创建css文件再导入HTML文件。导入方法<link rel=’stylesheet’ type=’css/text’ href=’css文件名路径’这样就可以把外部css文件样式导入内部。

5.可以把样式赋给一个值然后再把值赋给别人。

<style>

​ .ibm{

​ background-color: yellowgreen;

}

</style>

关于前端的一些粗糙的笔记但容易看懂，覆盖了HTML大部分基础对初学者很有参考价值。（把这些了解可以做点简单的网站了，当然如果要美观和丰富还要继续学习css和js啦）
